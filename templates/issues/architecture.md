# Architecture Issue (PDD)

## 1. Ziel
Welche architektonische Entscheidung oder Analyse ist notwendig?

## 2. Prüfbarkeit
- Welche Kriterien definieren eine gute Lösung?
- Welche Constraints müssen eingehalten werden?

## 3. Annahmen
- Domänenannahmen
- Systemgrenzen
- Nicht-Ziele

## 4. Verantwortung
- Welche Architekturentscheidungen triffst du selbst?
- Welche Alternativen darf die KI vorschlagen?

## 5. Kontext
ADR-Links, bestehende Architektur, Diagramme, technische Rahmenbedingungen.

## 6. Risiken
Welche langfristigen Konsequenzen hat die Entscheidung?

## 7. Artefakte
- ADR  
- Diagramme  
- Kriterienkatalog  

## 8. Done Criteria
- [ ] Entscheidung dokumentiert  
- [ ] Alternativen bewertet  
- [ ] Kriterien erfüllt  
- [ ] Risiken abgewogen
