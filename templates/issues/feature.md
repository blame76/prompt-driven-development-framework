# Feature Request (PDD)

## 1. Ziel
Was soll neu möglich sein?  
Für wen? Warum?

## 2. Prüfbarkeit
- Welche Kriterien definieren „Feature erfüllt“?
- Welche Szenarien müssen funktionieren?

## 3. Annahmen
- Welche Domänenannahmen gelten?
- Welche Grenzen sind bewusst gesetzt?

## 4. Verantwortung
- Welche Produktentscheidungen triffst du?
- Was darf die KI ausarbeiten?

## 5. Kontext
User Stories, Beispiele, bestehende Systeme, Constraints.

## 6. Risiken & Trade-offs
Welche Auswirkungen hat das Feature?

## 7. Artefakte
Welche Spezifikationen oder Modelle sollen entstehen?

## 8. Done Criteria
- [ ] Ziel erfüllt  
- [ ] Szenarien abgedeckt  
- [ ] Spezifikation vollständig  
- [ ] Risiken dokumentiert
