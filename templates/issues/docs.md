# Documentation Issue (PDD)

## 1. Ziel
Welche Dokumentation soll erstellt, verbessert oder ergänzt werden?

## 2. Prüfbarkeit
- Was macht die Dokumentation „gut genug“?
- Welche Fragen muss sie beantworten?

## 3. Annahmen
- Zielgruppe
- Vorwissen
- Scope

## 4. Verantwortung
- Welche inhaltlichen Entscheidungen triffst du?
- Was darf die KI formulieren oder strukturieren?

## 5. Kontext
Links, bestehende Dokumente, Beispiele.

## 6. Risiken
Welche Missverständnisse entstehen ohne diese Dokumentation?

## 7. Done Criteria
- [ ] Zielgruppe verstanden  
- [ ] Inhalt vollständig  
- [ ] Klarheit gegeben  
- [ ] Konsistenz mit PDD
